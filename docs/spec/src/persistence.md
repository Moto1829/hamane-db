# 永続化と耐久性

## 書き込みパス

```text
upsert / delete
  → WAL に追記 + fsync   ← ここで Ok が返る
  → memtable に反映       (以降、検索で見える)
  → (閾値超過で) フラッシュ: memtable → セグメント + HNSW
```

## 耐久性の保証

**`Ok` が返った書き込みは、直後にプロセスやマシンがクラッシュしても
失われません** (既定の `SyncPolicy::Always` の場合)。

- WAL の fsync が完了してから応答するため
- 再 open 時に manifest (フラッシュ済みデータ) + WAL リプレイ
  (未フラッシュ分) で完全に復元される
- WAL 末尾の書きかけレコード (クラッシュ痕跡) は安全に切り捨てられる。
  **確認応答済みのレコードが切り捨てられることはない**

`SyncPolicy::EveryN(n)` を使うと fsync 頻度が 1/n になり高速化しますが、
クラッシュ時に直近 (最大 n−1 件) の確認応答済み書き込みが失われる
可能性があります。

## クラッシュ一貫性

どの時点でクラッシュしても、再 open で以下が保証されます (テストで担保):

- 復元される状態は常に「操作列のある prefix」。半端な状態
  (レコードの一部だけ書かれた等) は決して見えない
- フラッシュ・コンパクションの途中でクラッシュした場合、
  未完成のセグメントや古い manifest の残骸は自動的に掃除され、
  WAL または直前の完全な世代から復元される
- ディスク破損 (CRC 不一致) は検出され、`Corrupted` エラーになるか
  破損前の prefix までが読み込まれる。**panic はしない**

## フラッシュ

memtable の内容を不変セグメントとしてディスクに書き出す操作。

- 契機: memtable が `flush_threshold_bytes` (既定 64 MiB) を超えたとき自動、
  または `db.flush()` / `col.flush()` で明示的に
- フラッシュは **DB 全体** (全 Collection) に対して行われる
- セグメントには行数が `hnsw_min_rows` 以上なら HNSW も構築・保存される
- フラッシュ後、反映済みの WAL は削除される

> ⚠️ v0.1 ではフラッシュ (HNSW 構築を含む) が書き込み呼び出しの
> スレッド上で同期実行されます。大規模な memtable のフラッシュ中、
> その Collection への書き込みはブロックされます
> ([制限事項](limits.md)、改善タスク todos/504)。

## コンパクション

複数セグメントを 1 個に統合し、上書き・削除された行を物理的に取り除いて
ディスクを回収する操作。

- 契機: フラッシュ後にセグメント数が `compaction_threshold` (既定 4) 以上に
  なったとき自動、または `db.compact()` で明示的に
- 統合後、削除マーカー (tombstone) は不要になるため破棄される
- **長時間の書き込みでもセグメント数とディスク使用量は有効データ量に
  収束する** (テストで担保)
- 実行中も検索は古いスナップショットで継続できる (ブロックしない)

## 復旧 (Database::open)

1. `CURRENT` が指す manifest を読み、CRC を検証
2. manifest に登録されたセグメントを mmap で開く
3. manifest より新しい WAL を順にリプレイして memtable を再構築
4. 不要ファイル (反映済み WAL、古い manifest、未完成セグメント) を掃除

復旧は WAL のサイズに比例した時間で完了します。定期的な `flush()` で
WAL を短く保つと再起動が速くなります。

## ディレクトリレイアウト

```text
<db_dir>/
  CURRENT                    # 有効な manifest 名
  MANIFEST-0000000042        # セグメント構成の世代
  wal/00000000000000000007.wal
  collections/<collection_id>/
    seg-000001/{vectors.bin, ids.bin, meta.bin, tombstones.bin, hnsw.bin}
    seg-000005/...
```

詳細は [オンディスクフォーマット](file-format.md) を参照。
