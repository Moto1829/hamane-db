# 202: WAL writer/reader とリプレイ

- Status: DONE (2026-07-12)
- Milestone: M2
- Depends: 201
- Design: docs/design/storage.md §2

## ゴール

書き込み操作を追記ログとして永続化し、末尾破損に耐えるリプレイを提供する。

## やること

- [ ] `WalRecord` enum (Upsert / Delete / CreateCollection / DropCollection) と
      フレーム encode/decode (crc32c + len + type + payload)
- [ ] `WalWriter`: `append(&WalRecord)`, `sync()`, `SyncPolicy { Always, EveryN(u32) }`
- [ ] `WalReader`: フレームを順に返すイテレータ。EOF・部分書き込み・CRC 不一致で
      停止し、停止位置を返す (呼び出し側が truncate する)
- [ ] ファイル命名 `wal/<seq:020>.wal` とディレクトリ走査 (seq 順ソート)

## 完了条件

- 書いたレコード列がそのまま読み戻せる
- 末尾を任意バイト数削ったファイルで、完全なフレームまでが読める
  (フレーム境界の全オフセットを機械的に試すテスト)
- フレーム途中の 1 バイト破壊で、そのフレーム以降が読まれない
